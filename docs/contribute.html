Contribute to Lightning

This project takes all contributions through pull requests only. The code should never be pushed directly to master.

The following guidelines apply to all contributors.

  • Found bug in Lightning? OR need some information that is missing OR interested to know more about Lightning? Just create the issue, so that the contributors can help you and provide you the necessary information. [Guidelines for Issues]
  • Please check for any existing Issues. Maybe it's already addressed. Duplicating of the issues, delays your time.
  • Be Clear about What problem is: What are the expected results? What happened instead? Any replication scenarios?
  • Use Gist or Pastebin to share your logs and provide the respective URL in the issue.
  • If you are adding any new feature or providing a bug fix, please make sure to pull the latest source code to your fork.
  • Give clear commit message.
  • If you’re able to patch the bug or add the feature yourself – Awesome! make a pull request with the code!
  • Slack us your pull request number.
