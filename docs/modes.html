Modes of Operation

Embedded Mode

In this mode, the Lightning-Core engine is started in memory. The client uses in memory core instance to execute URL.

Pros

  • Bypass the REST API calls between Lightning client and core, thereby saving network latency and bandwidth.
  • Saves management of Lightning-core instances.

Cons

  • The core runs in memory thereby ends up consuming JVM resources of the host application.
  • The client cannot use more than one core.
  • Not scalable horizontally.

Standalone Mode

In this mode, Lightning-Core instance is started as a standalone process. The client jar is added in the application that wants to use the lightning. The client is built from LightingClientBuilder and is used for various operations.

Pros

  • Highly Scalable
  • Distributed operation
  • Fault Tolerant

Cons

  • Need to start and maintain one or more external Lightning-core process(es).
  • The lightning client communicates with the core through REST API, thereby costing some network latency and bandwidth.
