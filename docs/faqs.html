1. Lightning-client or lighting client is not found in the maven repository?

This is how you can package and install lightning-client in the maven repository. These are two options. You can pick either one of the two.
Option 1: From Eclipse:
Step 1: Clone the lightning Project using below command and import the
Step 2: Import lightning-core and lightning-client projects into eclipse as maven project.
Step 3: Run maven clean install on both the projects

Option 2: From Command Line
Run these commands from the command line: 

>git clone git@github.corp.ebay.com:Lightning/lightning.git
>cd lightning-core
>mvn clean install assembly:single -DskipTests
>cd ../lightning-client
>mvn clean install -DskipTests

2. How is Lightning different from other async url executor libraries like asynchttpclient, apache httpClient, etc?

Other libraries are great but they are not distributed. Their peak performance is limited to resources available on a single machine. Lightning solves this problem by being distributed URL execution engine. So for scalability just adding more capacity to the core cluster is required.


3. How do I tweak and control various execution parameters?

Use LightningClient to update SystemConfig or Submit RequestConfig along with each request. Please take a look at javaDoc of RequestConfig and SystemConfig.


4. Does it supports any other languages than Java?

Lightning as of now supports only Java.


5. I used lightning and URL execution was not fast?

Make sure you submit once giving all the url task instead calling submit for each urlTask. Lightning handles queuing and batching internally and it works best if few bulk requests are given instead of multiple small requests.
